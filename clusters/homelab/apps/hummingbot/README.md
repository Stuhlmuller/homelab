# Hummingbot

Hummingbot runs as a persistent CLI trading client in the `finance` namespace.
This install follows the upstream Docker client shape, pins the
`hummingbot/hummingbot:version-2.14.0` image, and keeps the first rollout in a
manual trading mode: no exchange API keys, strategy configs, or live-trading
autostart settings are committed.

The client can place real orders after an operator connects exchange
credentials and starts a strategy. This repository intentionally does not
configure that path by default, and it does not guarantee profit.

## Runtime Contract

- Namespace: `finance`
- Image: `hummingbot/hummingbot:version-2.14.0`
- Persistent state: `hummingbot-conf`, `hummingbot-data`, `hummingbot-logs`,
  `hummingbot-certs`, `hummingbot-scripts`, and `hummingbot-controllers` PVCs
  on `nfs-default`
- Secret source: `/homelab/hummingbot/config-password` through the
  `hummingbot-config` ExternalSecret
- Network exposure: no Service, VirtualService, or public route

The config password is generated by the `IaC/live/aws-ssm-parameters`
Terragrunt stack. The pod uses it to create Hummingbot's
`.password_verification` file on first boot if the file is missing.

## Operator Access

Attach to the running client:

```sh
kubectl -n finance attach -it deploy/hummingbot -c app
```

Read recent logs:

```sh
kubectl -n finance logs deploy/hummingbot -c app --tail=100
```

Recover the config password from SSM when an operator needs to log in through
the CLI:

```sh
aws ssm get-parameter \
  --region us-west-2 \
  --name /homelab/hummingbot/config-password \
  --with-decryption \
  --query Parameter.Value \
  --output text
```

## Validation

Render local desired state:

```sh
kubectl kustomize clusters/homelab/apps/hummingbot
helm template hummingbot app-template \
  --repo https://bjw-s-labs.github.io/helm-charts \
  --version 4.4.0 \
  --namespace finance \
  --values clusters/homelab/apps/hummingbot/values.yaml
```

After sync, verify runtime objects:

```sh
kubectl -n argocd get application hummingbot
kubectl -n finance get externalsecret hummingbot-config
kubectl -n finance get deploy,pod,pvc -l app.kubernetes.io/instance=hummingbot
```

Expected result: `hummingbot-config` is ready, the deployment has one running
pod, and all six PVCs are bound.

## Live Trading Gate

Before enabling unattended trading, add a separate PR that includes:

- The strategy or script config as repository-owned desired state.
- Backtest and paper-trading evidence for the exact market pair.
- Exchange-side controls that disable withdrawals for any API key used by the
  bot.
- Rollback notes that preserve the Hummingbot config PVC unless the operator
  explicitly chooses to reset credentials.
